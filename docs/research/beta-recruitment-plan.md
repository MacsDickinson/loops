# Beta User Recruitment Plan

**Version:** 1.0
**Last Updated:** 2026-03-07
**Owner:** PM
**Target:** 20 qualified beta users for Week 9 launch

## Executive Summary

**Goal:** Recruit 20 qualified beta users for Discovery Loop Coach beta launch in Week 9

**Timeline:** Weeks 2-8 (6 weeks of recruitment before launch)

**Success Criteria:**
- 20 confirmed beta users by Week 9
- 70%+ match ICP (Ideal Customer Profile)
- 50%+ willing to provide weekly feedback
- Mix of user types (PM-led, engineer-led, solo builders)

## Ideal Customer Profile (ICP)

### Primary ICP

**Role:** Product Managers at software companies

**Company Size:** 50-500 employees

**Tech Stack:** Using AI coding tools (Cursor, GitHub Copilot, Replit, v0, Lovable)

**Pain Points:**
- Experiencing rework due to unclear specs
- AI tools generating code that doesn't match intent
- Friction between PM vision and engineering implementation
- Specs get outdated quickly, leading to misalignment

**Willingness to Pay:** $29/month subscription

### Secondary ICP (20-30% of beta)

**Role:** Technical founders, solo developers, engineering leads

**Company Size:** 1-50 employees (startups, indie hackers)

**Use Case:** Self-serve spec creation for AI-assisted development

**Value Prop:** Clarify thinking before prompting AI tools

## Recruitment Strategy

### Phase 1: Network & Direct Outreach (Weeks 2-4)

**Target:** 30 outreach → 10-12 signups

**Channels:**

1. **LinkedIn Direct Messaging**
   - Target PMs at 50-500 person companies with AI/tech in bio
   - Personalized messages (not mass spam)
   - Value prop: "Early access to tool that reduces AI coding rework"
   - CTA: 15-min call to discuss pain points + beta invite

2. **Product Manager Communities**
   - Product School Slack
   - Mind the Product community
   - PM HQ Discord
   - r/ProductManagement subreddit
   - Lenny's Newsletter community

3. **AI Coding Tool Communities**
   - Cursor Discord
   - GitHub Copilot users (LinkedIn search)
   - Replit community
   - v0.dev users
   - Lovable community

4. **Personal Network**
   - CEO's network (former colleagues, advisors)
   - PM's professional connections
   - Engineering team's PM contacts
   - Industry conference connections

**Message Template (LinkedIn):**

```
Hi [Name],

I noticed you're a PM at [Company] working with AI coding tools like [Cursor/Copilot].

I'm building Discovery Loop Coach – a tool that helps PMs create better specs for AI-assisted development. Early users are seeing 30% less rework.

Would you be interested in early beta access (free for 3 months)? Looking for 20 PMs to help shape the product.

Quick 15-min call to see if it's a fit? [Calendly link]

Best,
[PM Name]
```

### Phase 2: Content & Inbound (Weeks 3-6)

**Target:** 50 applications → 5-8 signups

**Channels:**

1. **Blog Posts**
   - "How to Write Specs for AI Coding Tools" (Medium, Dev.to)
   - "5 Reasons Your AI-Generated Code Doesn't Match Your Vision"
   - "Product Manager's Guide to GitHub Copilot"
   - Include CTA for beta access

2. **Product Hunt Pre-Launch Page**
   - Create "Coming Soon" page
   - Collect emails for beta access
   - Target "Upcoming" followers

3. **Twitter/X Campaign**
   - Share beta announcement thread
   - Use hashtags: #ProductManagement #AIcoding #BuildInPublic
   - Tag relevant communities and tools
   - Pin beta signup link to profile

4. **Hacker News**
   - "Show HN: Discovery Loop Coach – Better Specs for AI Coding" (Week 6)
   - Focus on technical audience (engineering-led PMs)
   - Include beta signup link

5. **Newsletter Sponsorships**
   - Lenny's Newsletter (PM audience)
   - TLDR (tech/dev audience)
   - The Pragmatic Engineer (engineering managers)
   - Budget: ~$500-1000 per newsletter

### Phase 3: Beta Application Funnel (Weeks 4-8)

**Target:** 100 applications → 5-10 signups (overflow for waitlist)

**Application Form:**

1. **Screening Questions**
   - What's your role? (PM, Engineer, Founder, Other)
   - Company size?
   - Which AI coding tools do you use? (Cursor, Copilot, Replit, etc.)
   - How often do you experience rework due to unclear specs? (Weekly, Monthly, Rarely)
   - Are you willing to provide weekly feedback during beta?

2. **Commitment Questions**
   - Can you commit to 2 hours/week for beta testing?
   - Would you pay $29/month after beta if the product delivers value?
   - What's your biggest pain point with current spec workflow?

3. **Contact Info**
   - Name, email, LinkedIn profile
   - Preferred communication channel (Slack, email, Discord)

**Auto-Reject Criteria:**
- "Rarely" experience rework
- Not using AI coding tools
- Not willing to provide feedback
- Outside ICP (e.g., non-technical roles)

**Manual Review:**
- 50-100 applications expected
- PM reviews top 30-40 candidates
- Prioritize: best ICP fit, high engagement signals, diverse use cases

1. Watch 2-min walkthrough: [Loom link]
2. Try the Product Coach persona

**Week 1 Check-In:**
- Email: "How's your first spec going?"
- Offer 1:1 support if stuck
- Share best practices from other beta users

**Weekly Feedback Survey:**
- What worked well this week?
- What was frustrating?
- Which features did you use?
- Would you recommend to a colleague? (NPS)
- Anything broken or buggy?

### Beta Success Metrics

**Week 10 (Launch + 1 week):**
- 70% complete first spec within 24 hours
- 50% day-2 return rate
- Specs average >500 words with acceptance criteria

**Week 11-12:**
- 70% week-1 return rate
- 30%+ report reduced rework
- 40%+ "very disappointed" if product disappeared (PMF signal)

**Feedback Cadence:**
- Weekly surveys (automated)
- Bi-weekly group call (optional, record for insights)
- Week 11: 1:1 interviews with 10-15 users

## Risks & Mitigation

### Risk 1: Insufficient Signups

**Mitigation:**
- Start outreach early (Week 2, not Week 6)
- Expand ICP to include engineering leads and solo developers
- Offer incentive: "Free for 6 months" instead of 3
- Lower barrier: No application form, just "claim your spot"

### Risk 2: Low-Quality Beta Users

**Mitigation:**
- Screen applications carefully (manual review)
- Prioritize users who are active in AI coding communities
- Look for engagement signals (detailed application answers)
- Replace inactive users after Week 1

### Risk 3: Low Engagement During Beta

**Mitigation:**
- Set clear expectations upfront (2 hours/week commitment)
- Weekly check-ins and reminders
- Gamify: "Top 3 most active users get 1 year free"
- Make feedback easy (1-click survey, not long forms)

### Risk 4: Negative Feedback Hurts Morale

**Mitigation:**
- Frame beta as learning opportunity, not product validation
- Celebrate learning, not just wins
- Separate "bugs" from "design decisions"
- Focus on PMF metrics, not individual complaints
